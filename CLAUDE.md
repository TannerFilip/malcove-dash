# malcove-dash — Threat Infrastructure Triage Dashboard

Personal SOC analyst tool for hunting malicious infrastructure (C2 panels,
stealer panels, AiTM kits, abusive RMM deployments, open directories, etc.)
via Shodan and Validin. Deployed entirely on Cloudflare's stack.

## What it does

1. Save Shodan/Validin queries and rerun them with **diff-on-rerun** — only
   new and changed hosts surface by default
2. Triage hosts through 7 states with full keyboard nav
3. Auto-enrich hosts (rDNS, full Shodan host detail) via a background queue
4. Nightly recheck of monitored/notable/reviewing hosts for cert rotations,
   JARM changes, etc.
5. Pivot from any host by cert serial, JARM, favicon hash, JA4X, or ASN+port
6. Passive DNS lookups via Validin proxy
7. Shodan quota dashboard (KV counter + live `/api-info`)

## Tech stack

| Layer | Choice |
|---|---|
| Hosting | Cloudflare Pages + Pages Functions |
| Frontend | Vite + React 18 + TypeScript + Tailwind + TanStack Query v5 + TanStack Router v1 |
| API | Hono mounted under `functions/api/[[path]].ts` |
| DB | D1 (SQLite) via Drizzle ORM + drizzle-kit migrations |
| Object storage | R2 (screenshots, archived banners — not yet actively used) |
| KV | Shodan quota counters |
| Background | Cloudflare Queues + Cron Triggers (separate Worker) |
| Auth | Cloudflare Access (GitHub SSO) — zero code in this repo |
| Validation | Zod everywhere at API boundaries |

## Database schema

Two migrations live in `src/db/migrations/`:
- `0000` — initial schema
- `0001` — adds `max_results` column to `queries`

Key design decisions:
- `hosts.id` = `"${ip}:${port}"` — natural dedup key, no surrogate needed
- `host_observations` is **append-only** — never update, always insert
- `bannerHash` = SHA-256 of JSON with keys sorted recursively (`sortKeysDeep`)
- All timestamps are **Unix seconds** (integer), never strings or milliseconds
- nanoid() for all other surrogate PKs

## Two-Worker setup

The project has two separate Cloudflare Workers:

**1. Pages Functions** (`wrangler.toml`)
- Serves the Hono API at `/api/*`
- Bindings: DB, KV, R2, ENRICHMENT_QUEUE
- Dev: runs as part of `npm run dev` (via wrangler pages dev)

**2. Enrichment Worker** (`src/workers/wrangler.worker.toml`)
- Queue consumer + cron handler
- Deploy separately: `wrangler deploy --config src/workers/wrangler.worker.toml`
- Dev: `wrangler dev --config src/workers/wrangler.worker.toml` (separate terminal)
- Test cron locally: `curl "http://localhost:8787/__scheduled?cron=0+2+*+*+*"`
- **`main = "./enrichment.ts"`** — path is relative to the config file's directory,
  NOT the project root. Don't change this to `src/workers/enrichment.ts`.
- Secrets must be set separately on this Worker:
  `wrangler secret put SHODAN_API_KEY --config src/workers/wrangler.worker.toml`

Cron schedule:
- `0 2 * * *` — enqueue stale hosts for re-enrichment
- `0 4 * * *` — recheck monitoring/notable/reviewing hosts for changes

## Gotchas discovered in development

### Vite / TanStack Router HMR
Changes to **route files** (`src/ui/routes/*.tsx`) often don't hot-reload
reliably in dev. After modifying a route file, do:
```bash
npm run build && npm run preview
```
Changes to hooks, components, and API files hot-reload fine.

### TanStack Router nested routes
`queries.$id.tsx` is automatically a **child** of `queries.tsx` in file-based
routing. `queries.tsx` uses a `QueriesLayout` component with `useRouterState`
to detect child route matches and render `<Outlet />` instead of the list.
If you add new nested routes, apply the same pattern.

### Shodan API response shapes
The search endpoint (`/shodan/host/search`) and the host detail endpoint
(`/shodan/host/:ip`) return **different shapes**:
- Search: each match is a single service — `data` is a **string** (raw banner)
- Host detail: `data` is an **array** of service objects
Two separate schemas exist: `ShodanSearchMatchSchema` and `ShodanHostSchema`.

### Shodan nulls vs undefined
Shodan sends `null` for missing optional fields, not `undefined`. All optional
string fields in the Shodan schemas use `.nullish()` not `.optional()`.

### D1 parameter limit
D1 allows max 100 bound parameters per statement. Drizzle doesn't warn you.
Bulk inserts must be chunked to ≤25 rows (4 columns × 25 = 100 params).

### Zod v4 boolean coercion
`z.coerce.boolean()` coerces the string `"false"` to `true` (because
`Boolean("false") === true`). URL boolean params use `z.string().optional()`
and are checked with `!== 'false'` in components.

### wrangler.worker.toml `main` path
`main` is resolved **relative to the config file's directory**, not the CWD.
So `main = "./enrichment.ts"` in `src/workers/wrangler.worker.toml` correctly
resolves to `src/workers/enrichment.ts`. Using `src/workers/enrichment.ts`
would produce a "file not found" error at deploy time.

### Queue consumer type signature
`@cloudflare/workers-types` expects `MessageBatch<unknown>` not
`MessageBatch<EnrichmentMessage>`. Cast inside: `const body = message.body as EnrichmentMessage`.

### TanStack Query v5 zero-arg mutations
If a `mutationFn` has an optional parameter, you must pass `undefined` explicitly:
`mutation.mutate(undefined)` — not `mutation.mutate()`.

### Validin API auth header
Uses `Authorization: BEARER <key>` (uppercase BEARER), not the usual `Bearer`.

## Code conventions

- **TypeScript strict**, no `any` — use `unknown` and narrow
- **Zod** for every API request body and every external API response
- **Unix seconds** for all timestamps (never ms, never ISO strings)
- **Drizzle types** are the source of truth — derive UI types via
  `InferSelectModel<typeof table>` re-exported from `src/shared/types.ts`
- **No default exports** except where a framework requires it
- **Error shape**: `{ error: { code: string, message: string, details?: unknown } }`
- **React**: functional components only, TanStack Query for server state,
  `useState`/`useReducer` for local UI state
- **Tailwind only** for styling — no CSS modules, no styled-components
- **nanoid()** for all generated IDs
